# HalalAvatar — мастер-план (СТАТУС: K1–K7 ВЫПОЛНЕНЫ)

Все запланированные этапы реализованы и проверены прогоном.
Этот документ теперь — описание принятых решений, а не задание.

## Что сделано

**K1 — движок каталога.** 17 клипов (7 новых с антиципацией, лагом и
овершутом), 10 луп-пресетов по 7200 мс, бесшовность доказана тестом
(кадр в t=0 побайтно равен кадру в t=D). `idle_sway2` с периодом 2×3600
даёт НОК 7200 с дыханием — иначе луп замыкался бы только через 47 секунд.
CLI пакетного рендера `render-catalog.mjs` (идемпотентный).

**K2 — студия.** Выбор анимации (10 чипов), `AvatarPlayer` с `loopMs`
(без зацикливания времени idle с endMs=7200 замирал после круга),
копирование конфига в буфер.

**K3 — браузерный экспорт.** `apps/web/src/lib/browser-render.ts`:
PNG-секвенция с настоящей альфой (canvas+JSZip, работает везде),
хромакейный MP4 через WebCodecs+mp4-muxer (быстрее реального времени,
с перебором профилей H.264), realtime-WebM как запасной путь.
Панель `DownloadPanel` с выбором кадра, формата и прогрессом.
Серверных мощностей не требует — файл собирается у пользователя.

**K5 — каталог.** Главная = сетка персонажей с постерами.
`/c/[id]` — живая SVG-анимация, выбор пресета, скачивание.
Пакетный рендер видео НЕ нужен для работы: превью анимируется вживую.

**K6 — мужской гардероб.** Хиджаб удалён из библиотеки. Добавлены:
тюбетейка, две причёски (ёжик, косой пробор), рубашка с воротником,
тоб, жилет, книга, чётки. Версия ассетов 1.1.0. Персонажей 9, все мужские.

**K7 — гигиена.** Next 15.5.4 (закрыта уязвимость), кнопка видео в чате
заменена ссылкой в студию (воркер не развёрнут), README переписан.

**Сверх плана:** кадрирования вынесены в `avatar-core/src/framings.ts`
как единственный источник истины — дублирование в воркере и браузере
уже начало расходиться. Тест-контракт `framings.test.ts` поймал реальный
баг: нечётную высоту 1419 у крупного плана, из-за которой yuv420p не
закодировался бы.

## Ключевые инварианты (не нарушать)

1. Контракты ядра неизменны: `samplePose`, `composeSceneSVG`, типы, LIMITS.
2. Пределы рантайма r ±25°, t ±40px, s ±0.05. Следствие: жесты «рука к лицу»
   и «рука над головой» недостижимы — не проектировать, лимиты не поднимать.
3. Клипы: `keys[0].t === 0`, ключи по возрастанию, id === имя файла.
4. Луп-пресеты: idle строго 0..D, D кратен периоду клипа, жест целиком в окне.
5. Безликость: никаких примитивов в лицевой зоне (круг 800,295 r62), кроме
   очков и одиночной skin-заливки. Валидатор отклонит.
6. Цвета только `var(--c-*)`.
7. После правки ассетов — `pnpm build:assets`; при новых частях bump version.
8. Фон остаётся в defaults (нужен для превью), но входит в BACKDROP_SLOTS
   и вырезается из всех экспортов.
9. Только мужские персонажи.

## Что осознанно НЕ делалось

Воркер на VPS и Redis (платно; браузерный экспорт закрывает потребность),
настоящая авторизация (сейчас общий демо-пользователь — до прихода
посторонних людей это нужно заменить), WebM с альфой (libvpx молча теряет
альфа-слой — проверено декодированием), деформация форм (изгиб рук),
lip-sync (лица нет).

## Возможные следующие шаги

- Авторизация перед публичным запуском.
- Расширение библиотеки: новые причёски, одежда, реквизит, палитры.
  Комбинаторика уже даёт тысячи сочетаний; узкое место — число архетипов.
- Новый архетип (в полный рост, другая поза) — самая дорогая работа:
  своя геометрия, пивоты, набор рук.
- Пакетный рендер каталога для ускорения загрузки страниц.

## K8 — Полный рост и профессии (ВЫПОЛНЕНО)

**Решение обёртки.** Новый архетип не потребовал параллельной библиотеки:
весь стек персонажа (голова, торс, руки со всеми пивотами) обёрнут в
`translate(224 28) scale(0.72)` через `Archetype.layout.wrap`. Каждая
существующая деталь, каждый клип и каждый пивот работают в полный рост
без изменений. Ноги и обувь — новые слоты `layout.underSlots`, рисуются
в координатах сцены ПОД обёрткой, бёдра подныривают под срез торса на
19px — шов закрыт (проверено пиксельным срезом). Пропорции: рост 705
единиц при голове 122 = 5.8 головы, взрослые.

**Гардероб профессий** (локальные координаты — работает в ОБОИХ
архетипах): костюм с галстуком, белый халат врача, фартук повара, худи,
каска, колпак повара, указка. Плюс слоты полного роста: брюки, длинный
тоб, обувь. Токены палитр расширены: --c-pants, --c-shoes (все 6 палитр,
белый список валидатора обновлён). Ассеты v1.2.0, 35 частей.

**Персонажи**: +6 стоячих профессий (Али-учитель с указкой, Фарид-врач,
Низами-строитель в каске, Анар-бизнесмен, Эмин-повар, Шейх в тобе до
пола). Всего 15, все мужские.

**Анимации «рассказывает»**: 5 новых клипов (weight_shift — луп переноса
веса с периодом 7200, explain_wide, point_up_R, palms_open_2, emphasize),
3 стоячих пресета (Рассказ, Приветствие стоя, Лекция) с полем
`archetypes` — build.ts валидирует. Всего 22 клипа, 13 пресетов,
бесшовность всех доказана тестом.

**Кадрирования** получили поле archetype: +fullbody (1080×1924),
fullbody_tight (760×1940), stand_square (1080×1080). Тест-контракт
переписан с границами фигуры на архетип. UI фильтрует пресеты и кадры
по архетипу; в студии переключатель «За столом / В полный рост»; CLI
выбирает постер-кадр и пропускает чужие кадрирования сам.

**Ограничение, принятое сознательно**: руки стоящего остаются в
«позе презентатора» (согнуты перед корпусом) — опущенные вдоль тела
руки требуют других локтевых пивотов, то есть пер-архетипных PIVOTS.
Для рассказывающего персонажа презентаторская стойка уместна.
